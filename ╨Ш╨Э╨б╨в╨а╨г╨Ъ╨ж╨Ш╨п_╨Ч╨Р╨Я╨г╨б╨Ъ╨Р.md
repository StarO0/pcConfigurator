# Полная инструкция запуска и тестирования v6.1

## 1. Требования

Установите Docker Desktop и запустите его. Отдельно устанавливать Python, Node.js,
PostgreSQL или Redis не нужно. На Windows используйте Docker Desktop с Linux containers.

Рекомендуется минимум 8 GB RAM и 10 GB свободного места для образов и базы.

## 2. Первый запуск на Windows

1. Полностью распакуйте ZIP в обычную папку, например `C:\Projects\PC-Configurator-v6.1`.
2. Не запускайте `.bat` прямо внутри ZIP.
3. Запустите Docker Desktop и дождитесь статуса `Engine running`.
4. Дважды нажмите `start-windows.bat`.
5. Первый запуск собирает образы и обычно занимает несколько минут.
6. Скрипт сам дождётся PostgreSQL, миграций, загрузки товаров, API и интерфейса.
7. Откройте <http://localhost:3000>.

Все команды в Windows launcher написаны ASCII-текстом: ошибки вида
`'здаю' is not recognized` и повреждённые кириллические команды из старой версии устранены.

Альтернативный запуск из PowerShell:

```powershell
cd C:\Projects\PC-Configurator-v6.1
Copy-Item .env.example .env -ErrorAction SilentlyContinue
docker compose up --build -d
docker compose ps
```

## 3. Первый запуск на Linux/macOS

```bash
chmod +x start-local.sh test-project.sh
./start-local.sh
```

## 4. Что происходит при запуске

Docker Compose запускает:

- `db` — PostgreSQL 15 с постоянным volume `postgres_data`;
- `redis` — очередь и кэш;
- `api` — FastAPI;
- `worker` — фоновые синхронизации;
- `beat` — расписание обновления цен;
- `frontend` — production Next.js.

API применяет Alembic до ревизии `77e6121a6d20`, загружает 332 готовых товара и запускает
quality cleanup. Незавершённые карточки старой версии удаляются. Если товар уже использован в
сохранённой сборке, он остаётся для истории, но становится неактивным.

Данные PostgreSQL переживают `docker compose down` и перезапуск компьютера.

## 5. Администратор

Откройте `Данные` и войдите:

```text
admin@pcbuilder.app
Local-admin-123
```

Для публичного размещения обязательно замените `SECRET_KEY`, `TOKEN_PEPPER`, пароль БД и
пароль администратора в `.env`.

## 6. Как добавляются новые товары

1. Откройте `Данные → Живое наполнение каталога`.
2. Выберите Morele, Komputronik, RTV EURO AGD или x-kom.
3. Проверьте `robots.txt` и условия сайта, затем поставьте галочку подтверждения.
4. Выберите размер пакета 10–100 страниц.
5. Нажмите `Обработать пакет`.

Парсер получает цену в PLN, наличие, фото и идентификаторы из магазина. Затем выполняется
keyless lookup Open Icecat:

- сначала по GTIN/EAN;
- если GTIN нет — по паре производитель + MPN;
- сохраняются Icecat ID, каноническая модель, главное фото, галерея и характеристики;
- предложение магазина остаётся отдельной записью с URL, ценой и временем проверки.

Quality gate не создаёт товар, если отсутствует хотя бы одно из следующего:

- цена больше нуля;
- HTTPS-фотография;
- поддерживаемая категория;
- производитель;
- GTIN/EAN либо связка производитель+MPN.

После первого подтверждённого запуска источник участвует в фоновом расписании каждые шесть
часов. Лимит и задержка защищают магазины от агрессивного обхода.

## 7. Как добавить другой магазин

В `Данные → Новый автоматический источник` заполните название, slug, базовый URL и URL
sitemap/feed. Выберите `Готовые карточки: магазин + Open Icecat`.

Поддерживаются:

- sitemap + Product/Offer JSON-LD;
- XML/YML, JSON и CSV feeds;
- HTML + CSS selectors;
- ручная очередь URL;
- JSON из Browser Collector.

Самый надёжный вариант — официальный XML/CSV/YML feed магазина с EAN, MPN, ценой, остатком
и URL карточки.

## 8. Проверка здоровья

```powershell
docker compose ps
docker compose logs --tail=100 api
docker compose logs --tail=100 frontend
docker compose logs --tail=100 db
```

Также можно запустить `CHECK_WINDOWS.cmd`. Успешная проверка подтверждает интерфейс, API и
наличие хотя бы одного готового товара.

## 9. Полный набор тестов

Windows:

```text
test-project-windows.bat
```

Linux/macOS:

```bash
./test-project.sh
```

Проверяются:

1. Docker build; lint и production build frontend выполняются внутри сборки;
2. PostgreSQL readiness и Alembic head;
3. 42 backend tests;
4. Ruff check и format check;
5. живой smoke test на PostgreSQL;
6. прокси `frontend → API` и готовый каталог.

## 10. Backup и восстановление

В панели `Данные`:

- `Скачать PostgreSQL backup` создаёт custom dump через `pg_dump`;
- `Восстановить backup` проверяет архив через `pg_restore --list`, затем восстанавливает его;
- после восстановления выполните `docker compose restart api worker beat`.

Командная строка:

```bash
docker compose exec -T db pg_dump -U pcbuilder -Fc pcbuilder > pcbuilder.dump
docker compose exec -T db pg_restore -U pcbuilder -d pcbuilder --clean --if-exists < pcbuilder.dump
```

## 11. Остановка и полный сброс

Обычная остановка сохраняет базу:

```bash
docker compose down
```

Полный сброс удаляет PostgreSQL и все пользовательские данные:

```bash
docker compose down -v
```

После следующего `start-windows.bat` снова загрузятся 332 стартовых товара.

## 12. Типовые проблемы

`docker is not recognized`: установите Docker Desktop и перезапустите терминал.

`Docker Desktop is not running`: откройте Docker Desktop и дождитесь запуска engine.

Порты 3000 или 8000 заняты: остановите другой процесс либо измените проброс
портов в `docker-compose.yml`.

`Failed to fetch` в браузере: выполните `docker compose ps`; `api` должен быть `healthy`.
Затем смотрите `docker compose logs --tail=200 api`.

Старая `.env` с неверными значениями: сравните её с `.env.example`. Путь стартового snapshot
принудительно задан в Compose, поэтому старая ссылка на файл больше не ломает запуск.

Магазин вернул 403/429: не отключайте robots/rate limits. Уменьшите пакет, увеличьте задержку
или используйте официальный feed этого магазина.
